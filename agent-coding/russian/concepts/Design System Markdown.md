# Design system markdown

## Кратко

Практика выражать design rules, references и UI intent в markdown- или text-артефактах, которые агенты могут потреблять напрямую.

## Текущий синтез

По мере сближения design agents и coding agents командам все чаще нужны текстовые артефакты, которые связывают visual intent и implementation behavior. Markdown references, llms.txt-files, design docs и структурированные design notes помогают агентам реализовывать UI с меньшей неоднозначностью и меньшим числом одноразовых prompt-ов.
