# Task-Completion Time Horizons of Frontier AI Models

## Сводка

Работа про time horizons — одна из самых ясных попыток измерить прогресс agentic coding в операционных терминах. Вместо вопроса о том, 'умна' ли модель, она спрашивает, какой длины реальную software-задачу агент может завершить с данной надежностью, что намного ближе к тому, как AI-Assisted Software Development-системы используются на практике.

## Ключевые тезисы

- Task horizon — более операционная метрика, чем общие benchmark scores.
- Frontier agents улучшаются на более длинных software-задачах.
- Пороги надежности важны не меньше, чем сырой capability.

## Практики и уроки

- Проектируйте workflows вокруг reliability envelopes, а не только вокруг peak demos.
- Ожидайте, что capability сначала будет улучшаться на средних по длине decomposable tasks.
- Используйте horizon framing, когда решаете, что делегировать.
