# The MCP Registry

## Сводка

MCP Registry превращает поддержку протокола в экосистему с discovery, metadata, namespaces и стандартной информацией для установки. Это важно, потому что рабочий extension-layer требует не только спецификацию, но и находимый каталог плюс модель provenance.

## Ключевые тезисы

- Зрелая экосистема MCP требует metadata и discovery, а не только protocol-compatibility.
- Namespace и execution-metadata — часть trust и operability.
- Инфраструктура registry меняет то, как команды выбирают и устанавливают coding-related MCP servers.

## Практики и уроки

- Предпочитайте discoverable servers с явной metadata вместо бессистемных install-snippets.
- Считайте provenance и execution instructions частью integration-review.
- Используйте registry-модель, чтобы отделить рост экосистемы от vendor lock-in.
