# OpenCode docs snapshot

## Сводка

OpenCode — хороший пример open-source lineage terminal agents: privacy-first, provider-flexible и явно ориентированный на multi-session work. Его flow с `/init` и AGENTS.md еще раз подчеркивает, что repo-local instruction files становятся стандартным интерфейсом во многих инструментах.

## Карта заметки

- `[[#Сводка]]`: что это за источник и почему он важен для AI-Assisted Software Development.
- `[[#Ключевые тезисы]]`: главные исторические, продуктовые или теоретические тезисы.
- `[[#Практики и уроки]]`: переиспользуемые операционные уроки.
- `[[#Связанные страницы]]`: какие концепты, инструменты и темы обновляет источник.

## Ключевые тезисы

- Open-source coding agents сходятся к repo-контрактам в стиле AGENTS.md.
- Гибкость по провайдерам — важный дифференциатор вне vertically integrated platforms.
- Параллельные сессии стали базовым ожиданием.

## Практики и уроки

- Используйте flows инициализации для посева repo-instructions.
- Эксплуатируйте несколько сессий для сравнения и фоновой работы.
- Предпочитайте privacy-preserving tools, когда важна локальность кода.
