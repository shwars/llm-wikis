# Introducing vibe design with Stitch

## Сводка

Stitch — это design-side counterpart для coding agents: захват намерения, high-fidelity iteration и параллельное исследование на agent-native canvas. Для AI-Assisted Software Development это усиливает тезис, что design documents становятся machine-readable control surfaces, а не статичными pre-engineering artifacts.

- `[[#Сводка]]`: что это за источник и почему он важен для AI-Assisted Software Development.

- `[[#Практики и уроки]]`: переиспользуемые операционные уроки.

## Ключевые тезисы

- Design intent теперь можно задавать на уровне goals и feeling, а не только frames.
- Параллельное исследование дизайна стало явным product-workflow.
- Дизайн-инструменты становятся частью того же agent stack, что и coding tools.

## Практики и уроки

- Храните design intent в переиспользуемом markdown и структурированных reference-файлах.
- Начинайте design prompts с goals и constraints, а не только с визуалов.
- Держите design- и implementation-artifacts тесно связанными.
