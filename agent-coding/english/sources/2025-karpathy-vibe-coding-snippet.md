# Karpathy on vibe coding

## Summary

The vibe-coding snippet marks the moment when a loose set of AI-assisted building habits became a named style. The phrase is partly a meme, but it also describes a real workflow: specify the outcome conversationally, let the model generate aggressively, and intervene mainly through prompts, checks, and taste.

## Key Claims

- Vibe coding is a cultural entry point into broader AI-Assisted Software Development, not the whole field.
- The style trades code intimacy for speed, iteration, and leverage.
- It works best for prototyping, exploration, and low-cost failure.

## Notable Practices

- Use vibe coding for exploration, not as a substitute for verification-heavy engineering.
- Expect maintenance debt if the human never reconstructs the system model.
- Pair fast prompting with later hardening and architecture work.
