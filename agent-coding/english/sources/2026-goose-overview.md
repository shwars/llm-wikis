# Goose overview readme snapshot

## Summary

Goose belongs to the broader local general-purpose agent cluster: desktop, CLI, API, many providers, and strong MCP integration for both coding and non-coding tasks. It matters because it sits exactly on the boundary the user asked to document.

## Key Claims

- The general-purpose local agent is becoming a serious product category next to coding agents.
- MCP is a major reason one agent can span coding, research, automation, and writing.
- Desktop, CLI, and API surfaces can share one extensibility model.

## Notable Practices

- Use one agent substrate across different surfaces when the extension layer is stable.
- Prefer open extension standards when the tool needs to cross coding and non-coding workflows.
- Treat coding as one high-value use case inside a broader workflow stack, not the only destination.
