# Introducing vibe design with Stitch

## Summary

Stitch represents the design-side counterpart to coding agents: intention capture, high-fidelity iteration, and parallel exploration on an agent-native canvas. For AI-Assisted Software Development, it reinforces that design documents are becoming machine-readable control surfaces instead of static pre-engineering artifacts.

## Key Claims

- Design intent can be specified at the level of goals and feeling, not just frames.
- Parallel design exploration is now an explicit product workflow.
- Design tools are becoming part of the same agent stack as coding tools.

## Notable Practices

- Store design intent in reusable markdown and structured references.
- Start design prompts from goals and constraints, not only visuals.
- Keep design and implementation artifacts tightly linked.
