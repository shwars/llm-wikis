# OpenCode docs snapshot

## Summary

OpenCode is a strong example of the open-source terminal-agent lineage: privacy-first, provider-flexible, and explicit about multi-session work. Its `/init` plus AGENTS.md flow also reinforces the idea that repo-local instruction files are becoming a standard interface across tools.

## Key Claims

- Open-source coding agents are converging on AGENTS.md-style repo contracts.
- Provider flexibility is a major differentiator outside vertically integrated platforms.
- Parallel sessions are now a baseline expectation.

## Notable Practices

- Use initialization flows to seed repo instructions.
- Exploit multiple sessions for comparison and background work.
- Prefer privacy-preserving tools when code locality matters.
