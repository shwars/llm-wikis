# Task-Completion Time Horizons of Frontier AI Models

## Summary

The time-horizon work is one of the clearest attempts to measure what agentic coding progress means in operational terms. Instead of asking whether models are 'smart,' it asks how long a real software task an agent can complete at a given reliability, which is much closer to how AI-Assisted Software Development systems are actually used.

## Key Claims

- Task horizon is a more operational metric than generic benchmark scores.
- Frontier agents are improving on longer-horizon software tasks.
- Reliability thresholds matter as much as raw capability.

## Notable Practices

- Design workflows around reliability envelopes, not only peak demos.
- Expect capability to improve first on medium-length decomposable tasks.
- Use horizon framing when deciding what to delegate.
