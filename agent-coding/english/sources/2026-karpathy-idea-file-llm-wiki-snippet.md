# Karpathy on the idea file and LLM Wiki

## Summary

The idea-file pattern pushes AI-Assisted Software Development one step further: instead of only distributing software or prompts, you distribute a structured description of the problem, constraints, references, and intended workflow so that agents can instantiate the artifact repeatedly. It is a design-document pattern for the agent era.

## Key Claims

- Intent artifacts can be more reusable than a specific generated app.
- Idea files and LLM wikis are durable coordination surfaces for agents.
- Knowledge-management structure is becoming part of software creation.

## Notable Practices

- Write portable problem files with goals, constraints, references, and quality bars.
- Prefer durable markdown artifacts over ephemeral prompt lore.
- Treat knowledge architecture as part of engineering architecture.
