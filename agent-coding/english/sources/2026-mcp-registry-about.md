# The MCP Registry

## Summary

The MCP Registry turns protocol support into an ecosystem with discovery, metadata, namespaces, and standard installation information. It matters because a usable extension layer needs more than a spec; it also needs a findable catalog and provenance model.

## Key Claims

- A mature MCP ecosystem requires metadata and discovery, not only protocol compatibility.
- Namespace and execution metadata are part of trust and operability.
- Registry-style infrastructure changes how teams evaluate and install coding-related MCP servers.

## Notable Practices

- Prefer discoverable servers with explicit metadata over unstructured install snippets.
- Treat provenance and execution instructions as part of integration review.
- Use the registry model to separate ecosystem growth from vendor lock-in.
