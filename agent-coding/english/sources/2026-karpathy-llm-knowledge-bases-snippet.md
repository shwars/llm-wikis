# Karpathy on LLM Knowledge Bases

## Summary

Karpathy's knowledge-base post reframes agent leverage around structured markdown knowledge systems. The same move that helps software agents can help research agents: put raw material in a stable repository, let the agent compile and maintain structured notes, and query the maintained corpus instead of re-reading the world every time.

## Key Claims

- Structured markdown knowledge stores can outperform ad hoc retrieval for moderate scales.
- Index files and summaries are first-class context artifacts.
- Agentic patterns transfer from codebases to research corpora.

## Notable Practices

- Store raw source material separately from synthesized wiki notes.
- Let agents maintain indexes, backlinks, and concept pages over time.
- Use markdown and filesystem structure as the knowledge interface.
