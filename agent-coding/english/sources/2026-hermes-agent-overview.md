# Hermes Agent overview readme snapshot

## Summary

Hermes Agent represents the self-improving branch of general-use agents: persistent memory, automated skill creation, many execution surfaces, and long-lived identity. It matters for this wiki because it makes the convergence between coding agents, agent memory, and non-coding work unusually explicit.

## Key Claims

- General-use agents are increasingly persistent and self-reflective rather than stateless chat wrappers.
- Skill creation can be part of the agent loop itself, not only manual authoring.
- Messaging, CLI, automation, and coding surfaces can share one agent identity.

## Notable Practices

- Treat memory, skills, and schedules as one operational system rather than separate features.
- Keep a clear distinction between background identity and per-task isolated work.
- Use subagents when a persistent agent still needs bounded parallel work.
