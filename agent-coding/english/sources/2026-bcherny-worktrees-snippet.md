# Boris Cherny on worktrees for parallel agents

## Summary

The Cherny worktrees snippet condenses one of the practical turning points in AI-Assisted Software Development: parallelism only becomes safe and pleasant when each agent gets isolated filesystem and branch state. Worktrees make multi-agent coding operational instead of theoretical.

## Key Claims

- Isolation is the foundation of reliable multi-agent coding.
- Worktrees are a practical answer to context and merge interference.
- Parallelism is valuable enough that power users run many agents at once.

## Notable Practices

- Use worktrees or equivalent isolation for concurrent agent tasks.
- Give each agent explicit ownership boundaries.
- Merge only after separate verification and human review.
