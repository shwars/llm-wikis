# OpenCrust overview readme snapshot

## Summary

OpenCrust is a compact, security-oriented reinterpretation of the always-on personal-agent pattern. It matters because it makes operational constraints like idle memory, startup latency, encrypted secrets, and local binding explicit product differentiators.

## Key Claims

- The general-use agent space is already differentiating on runtime and security posture, not only on model integration.
- Personal-agent platforms are starting to standardize around routing, schedules, MCP, and tool controls.
- Security defaults can themselves become a primary product surface for always-on agents.

## Notable Practices

- Treat encrypted credentials, sane bind defaults, and tool allowlists as first-class agent features.
- Measure the operational cost of always-on agents, not only feature breadth.
- Use comparative positioning to reveal what the market now expects from long-running agents.
