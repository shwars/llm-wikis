# Theses

## English Abstract

These are the most durable cross-source claims in the current AI-Assisted Software Development corpus.

- Verification is the strongest quality lever in modern AI-Assisted Software Development, stronger than prompt cleverness alone. See [[english/sources/2025-anthropic-claude-code-best-practices#Notable Practices|Claude Code best practices]] and [[english/sources/2025-openai-introducing-codex#Key Claims|Introducing Codex]].
- Structured repo-local knowledge beats giant one-off instruction blobs. See [[english/sources/2025-anthropic-effective-context-engineering#Key Claims|Effective context engineering]] and [[english/sources/2026-openai-harness-engineering#Key Claims|Harness engineering]].
- Design artifacts are becoming machine-readable control surfaces, not just handoff documents. See [[english/sources/2026-openai-harness-engineering#Summary|Harness engineering]], [[english/sources/2026-google-stitch-vibe-design#Summary|Stitch]], and [[english/sources/2026-anthropic-claude-design#Summary|Claude Design]].
- Multi-agent parallelism depends on isolation, ownership boundaries, and careful merge discipline. See [[english/sources/2026-bcherny-worktrees-snippet#Summary|Boris Cherny on worktrees]] and [[english/sources/2026-openai-introducing-the-codex-app#Key Claims|Introducing the Codex app]].
- The best current harnesses treat the repository as a [[english/concepts/Repository as System of Record|system of record]] for plans, constraints, and workflows. See [[english/sources/2026-openai-harness-engineering#Summary|Harness engineering]] and [[english/sources/2026-opencode-docs#Summary|OpenCode docs]].
- Coding agents already generalize to non-coding tasks such as deep research, note-taking, release briefs, issue triage, and visual QA. See [[english/sources/2025-anthropic-building-agents-with-claude-agent-sdk#Summary|Claude Agent SDK]] and [[english/sources/2026-openai-introducing-the-codex-app#Summary|Introducing the Codex app]].
- Vibe coding is real and productive, but it is only one low-friction style inside a broader field. See [[english/sources/2025-karpathy-vibe-coding-snippet#Summary|Karpathy on vibe coding]] and [[english/themes/Agentic Coding and Vibe Coding|Agentic Coding and Vibe Coding]].
- Task slicing and explicit done conditions are universal requirements for high-autonomy tools such as [[english/tools/Devin|Devin]], [[english/tools/Codex|Codex]], and [[english/tools/GitHub Copilot Coding Agent|GitHub Copilot Coding Agent]]. See [[english/sources/2026-devin-intro#Notable Practices|Introducing Devin]], [[english/sources/2025-github-copilot-coding-agent-ga#Notable Practices|Copilot coding agent GA]], and [[english/sources/2025-openai-introducing-codex#Notable Practices|Introducing Codex]].
- State of the art is improving on longer tasks, but reliability still matters more than benchmark theater. See [[english/sources/2026-metr-time-horizons#Summary|METR time horizons]] and [[english/sources/2026-metr-productivity-transcripts#Summary|METR productivity transcripts]].
- The long-term direction is toward agentic software creation plus knowledge-system maintenance, not just better autocomplete. See [[english/sources/2026-karpathy-llm-knowledge-bases-snippet#Summary|Karpathy on LLM Knowledge Bases]] and [[english/sources/2026-openai-codex-for-almost-everything#Summary|Codex for (almost) everything]].
- Coding agents and [[english/analyses/General Use Agents - 2026|general-use agents]] are converging through memory, schedules, [[english/concepts/Computer Use|computer use]], [[english/concepts/Agent Skills|skills]], and live integrations. See [[english/sources/2026-hermes-agent-overview#Summary|Hermes Agent overview]], [[english/sources/2026-openclaw-overview#Summary|OpenClaw overview]], and [[english/analyses/General Use Agents - 2026|General Use Agents - 2026]].
- [[english/concepts/Model Context Protocol|MCP]] and [[english/concepts/Agent Skills|Agent Skills]] are complementary extension layers: one brings live systems in, the other packages reusable procedures. See [[english/sources/2026-mcp-what-is-mcp#Summary|What is MCP?]], [[english/sources/2026-anthropic-claude-code-skills#Summary|Claude Code skills]], and [[english/analyses/MCP vs Agent Skills|MCP vs Agent Skills]].
