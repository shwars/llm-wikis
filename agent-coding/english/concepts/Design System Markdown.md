# Design System Markdown

## English Abstract

The practice of expressing design rules, references, and UI intent in markdown or text artifacts that agents can consume directly.

## Current Synthesis

As design agents and coding agents converge, teams increasingly need textual artifacts that bridge visual intent and implementation behavior. Markdown references, llms.txt files, design docs, and structured design notes help agents implement UI with less ambiguity and fewer one-off prompts.
