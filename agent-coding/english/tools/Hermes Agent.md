# Hermes Agent

## English Abstract

A self-improving, persistent general-use agent that combines memory, skills, schedules, messaging, and coding-capable work.

## Current Synthesis

Hermes Agent sits at the intersection of persistent personal assistance and coding-capable agent infrastructure. Its distinctive move is to treat memory and skill creation as part of the agent loop itself. That makes it especially relevant to this wiki: Hermes is one of the clearest examples of coding-agent ideas expanding into a long-lived general-use system.

## Tool Profile

- `Primary surface`: CLI, messaging gateway, scheduled work, and server-hosted sessions.
- `Target use case`: persistent personal or team agent workflows across coding, research, and automation.
- `Autonomy model`: long-running agent with background work, schedules, and subagent delegation.
- `Context and memory`: cross-session memory, skill reuse, and self-improvement loops.
- `Extensibility`: native skills, hubs and registries, MCP client mode, and MCP server mode.
- `Security and ops`: strong configurability with explicit exposure control, but requires disciplined operations.
- `Coding relationship`: closer to coding than OpenClaw because it directly embraces subagents, coding skills, and tool composition.
