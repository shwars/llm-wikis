# Memoh

## English Abstract

A container-first, multi-bot, always-on agent platform that combines persistent memory, channels, skills, and MCP.

## Current Synthesis

Memoh pushes the general-use agent pattern toward multi-bot operations and admin infrastructure. It matters because it looks less like a chatbot and more like a hosted agent platform with isolation, memory, channels, skills, and MCP under one roof. That brings it close to the concerns of serious coding-agent operations even though it is not a coding-first product.

## Tool Profile

- `Primary surface`: web UI, channels, containerized bot runtime, and admin config.
- `Target use case`: always-on multi-bot automation with persistent context and explicit isolation.
- `Autonomy model`: long-running containerized agents with channel and tool access.
- `Context and memory`: persistent memory and per-bot filesystem or environment boundaries.
- `Extensibility`: MCP, skills, slash commands, and model or channel configuration.
- `Security and ops`: isolation and graphical administration are part of the core value proposition.
- `Coding relationship`: useful as a model for teams that want agent infrastructure around coding-adjacent work rather than only editor assistance.
