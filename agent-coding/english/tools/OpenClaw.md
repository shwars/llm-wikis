# OpenClaw

## English Abstract

A local-first, always-on personal agent platform built around channels, a gateway, skills, and persistent state.

## Current Synthesis

OpenClaw is not a coding-first IDE assistant. It is an always-on personal agent stack that happens to overlap with coding-adjacent automation through workspaces, routing, skills, MCP-compatible integrations, and persistent sessions. For this wiki, it is important as the clearest claw-style example of where coding-agent patterns meet general-use personal agents.

## Tool Profile

- `Primary surface`: local gateway, messaging channels, voice, and CLI.
- `Target use case`: personal assistance, long-running automations, message-driven workflows, and agent routing.
- `Autonomy model`: long-lived, event-driven, always-on operation with scheduling and per-agent routing.
- `Context and memory`: persistent sessions, local workspace state, and agent-specific instructions.
- `Extensibility`: OpenClaw skills, scoped skill loading, and MCP-style external integrations.
- `Security and ops`: powerful but high-risk; host, credential, and skill hygiene are central.
- `Coding relationship`: indirect but growing through triage, automation, repo-adjacent tasks, and delegated utility work.
