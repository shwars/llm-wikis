# Goose

## English Abstract

A local desktop and CLI general-purpose agent that spans coding, research, writing, automation, and data work.

## Current Synthesis

Goose is the clearest boundary case in this extension: a local general-purpose agent that comfortably includes coding but is not defined by coding alone. It matters because it shows how MCP and local agent surfaces let one product straddle software work, research, writing, and automation without becoming an always-on personal assistant.

## Tool Profile

- `Primary surface`: native desktop app, CLI, and API.
- `Target use case`: local general-purpose work across code, research, automation, writing, and data analysis.
- `Autonomy model`: interactive local agent rather than a messaging-native always-on assistant.
- `Context and memory`: local runtime context with extension-driven reach into external systems.
- `Extensibility`: strong MCP emphasis plus multi-provider support.
- `Security and ops`: local-first deployment and open-source extensibility are core selling points.
- `Coding relationship`: direct and strong, but coding is framed as one major workflow among several.
