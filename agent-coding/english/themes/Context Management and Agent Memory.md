# Context Management and Agent Memory

## English Abstract

This theme tracks the shift from prompt engineering toward structured context engineering and durable memory artifacts.

## Current Synthesis

Current AI-Assisted Software Development depends on managing attention budgets, not just writing clever prompts. The frontier pattern is to store stable repo knowledge in AGENTS.md or CLAUDE.md-style files, move long-lived design or domain knowledge into markdown documents, compact sessions when they bloat, and spawn isolated subagents when exploration would otherwise poison the main context. Karpathy's LLM Wiki and idea-file patterns extend the same logic to research and knowledge work. Skills add a progressive-disclosure layer on top of this: reusable procedural context that loads only when needed. MCP adds the complementary live-context layer: external systems that can be queried or acted on without pasting their contents into the transcript.
