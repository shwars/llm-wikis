# Verification, Testing, and Automatic Debugging

## English Abstract

This theme tracks the feedback loops that keep AI-Assisted Software Development from collapsing into plausible but wrong output.

## Current Synthesis

The clearest consensus in current AI-Assisted Software Development is that verification dominates prompt cleverness. Strong agents improve most when they can run tests, inspect screenshots, compare outputs, and retry automatically. Automatic debugging is therefore not a minor convenience but the central creation loop: propose, execute, observe failure, patch, and re-run until the verification signal is green or the agent escalates uncertainty.
