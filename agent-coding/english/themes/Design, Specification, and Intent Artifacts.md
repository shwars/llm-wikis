# Design, Specification, and Intent Artifacts

## English Abstract

This theme covers the files and artifacts that tell an agent what good work looks like before it edits code.

## Current Synthesis

AI-Assisted Software Development works best when intent is externalized into durable artifacts: product specs, design docs, architecture notes, design-system references, reliability constraints, and idea files. The central shift is that these documents are no longer only for humans. They are machine-readable control surfaces that shape search, planning, implementation, and evaluation. Design tools like Stitch and Claude Design strengthen this trend by making visual intent easier to generate and hand off.
